## Кеширование 

Кеширование - сохранение данных в промежуточном буфере
для оперативного доступа

В кеш принято сохранять те данные, которые вероятнее всего 
будут запрошены пользователем в следующий раз. 

### 2 типа кеша 

Клиентский кеш предназначен для сохранений статической информации 
на стороне клиента (браузера). Например: стили, JS, изображения, видео,
аудио. 

Серверный кеш предназначен для хранения динамической информации,
получаемый на основе запросов пользователя. Например: выборки из БД, 
вычисляемые значения. 

### Настройка 

Основная настройка подключения кеша для проекта - настройки базы, которая 
включает только бэкенд и расположение. 

В качестве бекенда можно использовать: 
- нереляционные БД,
- реляционные БД, 
- файлы,
- заглушки для тестирования функционала. 

### Брокер 

Брокер - посредник между проектом и вспомогательной функциональностью. 

Брокером функционала кеширования будет хранилище кеша: файл или любая БД. 

Примеры брокеров: 

- Таблица в реляционной БД
- Нереляционная БД (Reddis, Memcached)
- Файлы

### Настройка 

1. Установить пакет, который будет работать с брокером. На него мы будем ссылаться в разделе Backend
2. Настроить в файле settings.py подключение к кешу, указав подходящий бекенд и расположение брокера
3. Подключение к брокеру может быть закрыто настройками авторизации
